# 历史金融图谱

这是一个关于金融历史、经济机制、货币制度、战争冲击和市场行为的个人知识库。

它不是通用历史百科，也不是新闻剪藏夹。它的目标是把过去几百年里重要国家和地区经历过的金融危机、繁荣周期、战争冲击、政策选择和资产表现，整理成一个可以反复查询、比较和复盘的“市场记忆系统”。

---

## 为什么建立这个项目

学习历史，是为了避免未来重复相同的错误。

对我来说，这个知识库的意义不只是投资，也是一种长期的认知建设。钱会贬值，资产会波动，但一个人理解历史、理解风险、理解政策和市场机制的能力，可以长期积累，也可以传给下一代。

这个项目希望帮助我回答：

- 过去重要金融事件到底发生了什么？
- 风险是怎样从一个部门传导到另一个部门的？
- 政府、央行、银行、企业和居民分别做了什么？
- 股票、债券 bonds、房地产 real estate、汇率 currencies、商品 commodities 和现金 cash 在不同环境中如何表现？
- 今天的世界像过去哪个阶段？相似在哪里，不同在哪里？
- 哪些历史经验可以用于风险控制、资产保护和长期判断？

---

## 这个知识库用来做什么

核心用途有四个：

| 用途 | 说明 |
|---|---|
| 记录历史事件 | 把重大金融危机、战争冲击、泡沫和政策转折整理成结构化笔记。 |
| 提炼机制 | 从事件中抽出信贷、杠杆、流动性、通胀、汇率、资产负债表等可复用机制。 |
| 比较案例 | 用相同维度比较不同国家、不同时代和不同危机。 |
| 训练判断 | 帮助自己看懂时代位置、风险积累、政策空间和资产表现。 |

这个库不追求“什么都收”，而追求每篇笔记都能服务未来推理。

## 如何使用

最小使用流程：

```text
提出一个问题
→ 搜索已有笔记
→ 判断材料类型
→ 使用对应模板
→ 先写事实
→ 写出机制链条
→ 链接相关国家、资产、概念、模式和可对比事件
→ 标记来源状态和不确定性
→ 提炼可复用经验
```

常用入口：

- [[Project Vision（项目愿景）|项目愿景]]：说明这个知识库为什么存在。
- [[How to Use This Vault（如何使用这个知识库）|如何使用这个知识库]]：说明日常怎么查、怎么写、怎么维护。
- [[Research Workflow（研究流程）|研究流程]]：说明从问题到笔记、从事实到机制的研究步骤。
- [[Writing Rules（写作规则）|写作规则]]：说明写作格式、来源要求、标签和链接规则。

## 写作原则

- 机制优先于叙事。
- 事实优先于观点。
- 结构优先于长篇散文。
- 区分事实、机制、解释、不确定性和投资启发。
- 不把没有来源的内容写成确定结论。
- 不随意创建重复笔记。
- 用链接表达关系，用标签分类，用 YAML 让笔记可查询。

## 未来希望

这个知识库最终希望变成一个长期可用的金融历史图谱：

- 能按国家、年代、事件、资产和机制反复查询。
- 能比较不同危机、不同政策周期和不同资产表现。
- 能帮助我理解今天处在什么历史位置。
- 能在未来教育孩子时，提供一套分析世界、理解风险、看待资产的框架。
- 能让一个普通家庭逐步积累自己的金融记忆，而不是每一代都从零开始。

最终目标不是预测每一次涨跌，而是建立一种长期判断力：

> 看懂历史如何重演，也看懂每一次“相似”背后的不同。
